Quality: About This Data Set

Data quality

The dataset was not complete as many parameters were missing.

For many samples, the activity_type values were missing. For all cases ,the label values of location table are missing.

The accuracy of collected data is not high as expected.

The accuracy value of location table is very high (highest values over 1000m). The confidence values of activity table are all 2 (range 1-100) thus the confidence of activity values can't be determined.

To assure the quality of data, although what can be done is limited, here's what I did when processing the datesets:

Activity table:

Lines where activities = “” were dropped; Column device_id is dropped since only one device was used; Column confidence is dropped since all confidence is 2. There might be a bug in the APP.

Locations table:

Rows with accuracy > 1000 (m) were dropped; Column device_id is dropped since only one device was used; Column double_bearing is dropped since all values are -1; Column label is dropped since all values are NULL; Column provider is dropped since all values are gps.

Overall, although the many parts of dataset are inconsistent and imcomplete, the final visualization was still able to provide some insights.

Visualization design choice

For visualization, I decided to left join activity table to locations table on timestamp so that each dot represent a location where I have been to and what my activity is at that spot. The color of the dot was set based on the activity type so that the viewers can visualize the activity at each location without clicking into it. Tooltip was set to contain detailed information for each dot including datatime, activity type, latitude, longitude and speed. An open source WMS terrestris was set as the background of map so that viewers can better visualize the locations I have been to.
